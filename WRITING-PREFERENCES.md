# Writing preferences

## Style rules

- Use British English spelling
- Use active voice
- Write short, declarative sentences. Vary length to avoid monotone.
- Prefer periods over commas for clarity
- Avoid repeating the same words in a paragraph. Use synonyms or rephrase.
- Be direct, specific, and concrete. Strip corporate jargon.
- Back claims with evidence, examples, or metrics
- Explain technical terms in plain language
- Use contractions for a warmer tone
- Use positive phrasing. Say what something _is_ rather than what it _isn't_.
- Say "you" more than "we" for external audiences

## Titles

- Make a clear promise so readers know what they get
- Tap into controversial points the audience holds but back them up with data. Avoid clickbait.
- Share something uniquely helpful

## Word choices

**Remove entirely**: a bit, a little, actually, agile, arguably, battle tested, business logic, disruptive, game-changing (use specific benefit), great (be specific), innovative, just, modern, out of the box, performant (use "fast and reliable"), pretty, quite, rather, really, very, that (often removable)

**Replace with**:

| Avoid | Use |
|-------|-----|
| assistance | help |
| attempt | try |
| best practice | proven approaches |
| commence | start |
| delve | go into |
| facilitate | help or ease |
| implement | do |
| initial | first |
| leverage | use |
| mission-critical | important |
| numerous | many |
| referred to as | called |
| remainders | rest |
| robust | strong |
| seamless | automatic |
| utilise | use |
| webinar | online event |

## Banned phrases

- "I think", "I believe", "we believe": state directly
- "it seems", "sort of", "kind of": remove
- "a lot", "a little": be specific
- "We obsess over ___": remove
- "We're excited", "Today, we're excited to": use "We look forward"

## Avoid LLM patterns

- Replace em dash (—) with a colon, semicolon, or period. Never use an em dash.
- Skip openers like "Great question!", "You're right!", "Let me help you", "Let's dive into..."
- Avoid cliches: "In today's fast-paced digital world", "In the ever-evolving landscape of", "it's not just [x], it's [y]"
- Skip self-referential disclaimers: "As an AI", "I'm here to help you with"
- Skip closers: "In conclusion", "Overall", "To summarise", "Hope this helps!"
- Avoid overusing "Furthermore", "Additionally", "Moreover"
- Avoid hedge words unless uncertainty is real: "might", "perhaps", "potentially"
- Don't stack hedging: "may potentially", "it's important to note that"
- Don't create symmetrical lists starting with "Firstly...Secondly..."
- Prefer bullets over numbered lists when order doesn't matter
- Use sentence case headings
- Use straight quotes and ASCII punctuation. Remove smart quotes ("), em-dashes, non-breaking spaces.
- Use ' instead of `
- Delete empty citation placeholders like "[1]" with no source

## Punctuation

- Use Oxford commas consistently
- Use exclamation points sparingly
- Sentences can start with "But" and "And", but don't overuse
